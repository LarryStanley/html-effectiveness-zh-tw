Research & Learning · 功能摘要

acme/api 的速率限制是怎麼運作的

一句話重點（TL;DR） — 每個請求都會過一層 rateLimit() 中介層。它會把呼叫方 解析成一個 桶金鑰（bucket key），去 Redis 撈對應的權杖桶（token bucket），接著要嘛扣 掉一個 token，要嘛直接回 429。速率限制是按路由在 config/limits.yaml 裡設定的；沒有特別設定的路由，就會套用 default 這組預設值（每支 API 金鑰每分鐘 100 次請求）。

請求流程，一步一步來

展開每個步驟，看看背後跑了什麼、程式碼在哪裡。整段流程大概只有 40 行，會給每個請求加上約 0.4 ms 的 p50 延遲。

1 · 確認呼叫方 middleware/ratelimit.ts:21

中介層會先把請求歸納成一組 bucketKey：如果帶了 Authorization 標頭就用 API 金鑰；沒有的話，就退而求其次看客戶端 IP（靠 x-forwarded-for 鏈，但我們只信任自家 LB）。匿名 IP 流量會套用嚴格許多的預設限制。

2 · 查詢權杖桶 lib/tokenBucket.ts:9

路由名稱加上 bucket key 會對應到一組 Redis hash（rl:{route}:{key}），裡面存了 tokens 和 updatedAt。如果這組 key 不存在，系統會直接懶載入建一個 滿水量的桶子給你，完全不需要暖機。

3 · 補水並扣除 lib/tokenBucket.ts:31

系統會根據經過的時間計算補水量（rate × Δt，上限是 burst），然後扣掉 一個 token。整段讀取、修改、寫回的過程全包在一個 Lua script 裡頭跑，所以就算同時有多個請求 進來，也不會發生重複扣款的問題。

4 · 額度空了就拒絕 middleware/ratelimit.ts:48

如果 script 算出來 tokens < 0，中介層就會直接短路，回一個 429 Too Many Requests，並且把 Retry-After 設成距離下次補滿一個 token 所需的秒數。而成功的回應則一定會帶上 X-RateLimit-Remaining。

幫你的路由加上速率限制

你完全不用動到中介層的程式碼。只要在 config/limits.yaml 裡用路由名稱當 key 加一筆 設定，然後（可選）在路由上標個 tag 讓中介層找得到就好。

# config/limits.yaml
default:
  rate: 100/min
  burst: 120

search.query:
  rate: 20/min
  burst: 40
  key: api_key        # 或: ip
// routes/search.ts
router.post(
  "/search",
  rateLimit("search.query"),
  handler,
);
HTTP/1.1 429 Too Many Requests
Retry-After: 17
X-RateLimit-Limit: 20
X-RateLimit-Remaining: 0

{ "error": "rate_limited", "retry_after": 17 }
★
如果你只需要預設的限制，其實連 YAML 都不用加。直接把 handler 包進沒有帶參數的 rateLimit() 裡就好，路由名稱會自動從路徑推斷出來。

值得知道的陷阱/眉角

  • 在 dev 環境中，限制是 per-process 的。 如果沒有設定 REDIS_URL，Redis client 會 fallback 到 in-memory map，所以本機測試沒辦法 反映真實叢集的行為。
  • Burst 不等於 rate。 burst 是桶子容量；如果 呼叫方閒置了一分鐘，就算 rate 很低，他還是可以在一瞬間打出 burst 數量的請求。
  • 串流回應只算一次。 Token 是在請求開始時就扣掉了；所以一個跑 30 秒的 SSE 串流， 也只會耗用一個 token。

FAQ

我要怎麼讓內部流量不受限制？
呼叫方加上 x-acme-internal: 1 標頭；中介層會拿這個值去對比 mTLS peer name， 確認沒問題就直接跳過權杖桶。
我要去哪看誰被限制住了？
每次出現 429 都會發出一個 ratelimit.rejected metric，上頭帶有 路由和 key 類型的 tag。在 API → Health 那邊有對應的 Grafana 面板可以看。
可以給單一使用者更高的限制嗎？
可以，在 YAML 裡的 overrides: 底下加上他們的 API 金鑰就行。調整 overrides 不需要重新部署就能生效。
