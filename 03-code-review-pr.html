這個 PR 做了什麼

  • 把 TaskList 裡面那套「送出請求、等回應、再重新抓資料」的做法，換成直接寫進樂觀快取。這樣不管是要切換狀態還是重新排序，操作起來都會有秒開的感覺，不用再等大約 300ms 的來回時間。
  • 新增了一個小巧的 useOptimisticTasks hook。它把更新操作包起來，會先快照目前的列表，等出錯時再自動復原。
  • 改了 API client，讓每次更新都能帶上冪等金鑰（idempotency key），而且如果觸發了自動復原，會跳出 toast 提示。
