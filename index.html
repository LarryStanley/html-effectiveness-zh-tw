部落格文章的配套

HTML 超乎尋常的有效性

這是代理（agent）產生的二十個獨立 .html 檔，用來取代整片 markdown 文字牆。每一份都把那種你只會掃過的文件，換成你真的會讀的文件；直接用瀏覽器打開任何一個就行。以下依它們取代的工作類型分類。

01

探索與規劃

3 個範例

還不確定自己要什麼的時候。請代理一次展開好幾個方向，並排擺在一起讓你直接指著其中一個說「就要這個」，而不是連讀三面文字牆、還得全部記在腦裡。選定之後，再把選中的方向寫成實作者看得懂的計畫。

三種程式做法
把同一個問題的三種解法並排比較，取捨直接標註在旁邊。
01-exploration-code-approaches.html→
視覺設計方向
幾組版面與配色選項實際渲染出來，讓你直接看到、而不是憑空想像。
02-exploration-visual-designs.html→
實作計畫
時間軸上的里程碑、資料流程圖、內嵌的模擬畫面、風險最高的程式碼，再加上一張風險表；這就是你要交辦出去的那份計畫。
16-implementation-plan.html→
02

程式碼審查與理解

3 個範例

diff 與呼叫圖（call graph）是空間資訊，markdown 會把它們壓扁。讓代理把改動渲染成帶註解的 diff、把模組畫成方塊與箭頭，或寫出審查者真正想看的拉取請求（PR）說明，讓程式碼的結構一眼就看得出來。

附註解的拉取請求
加上邊界註記、嚴重程度標籤與跳轉連結的 diff，比在終端機裡捲動好掃多了。
03-code-review-pr.html→
給審查者的 PR 說明
作者這一方：動機、前後對照、逐檔走一遍並說明為什麼這樣改，以及審查時該把重心放在哪。
17-pr-writeup.html→
模組地圖
把陌生的套件畫成方塊與箭頭，標出熱門路徑（hot path）並列出進入點。
04-code-understanding.html→

設計

2 個範例

HTML 就是設計系統（design system）交付時所用的媒介，所以拿來談它再自然不過。設計權杖（token）變成色票，元件變成排滿整頁的總覽，而這份成品可以直接餵回下一個提示詞（prompt）。

活的設計系統
從 repo 拉出顏色、字級與間距權杖，渲染成可複製的色票。
05-design-system.html→
元件變體
把同一個元件的每種尺寸、狀態與意圖，全排在同一頁方便檢視。
06-component-variants.html→

原型製作

2 個範例

動態與互動沒辦法用描述的，只能實際去感受。一個用完即丟的頁面，帶上真實的緩動曲線或真實的點擊流程，五秒鐘就能告訴你一整段文字永遠說不清的事。

動畫沙盒
把轉場單獨抽出來，配上時長與緩動的滑桿，讓你接線前就能先調好。
07-prototype-animation.html→
可點擊的流程
四個畫面串在一起，擬真度足以讓你感受這個互動對不對。
08-prototype-interaction.html→

插圖與圖表

2 個範例

內嵌 SVG 等於給代理一支真正的筆。跟它要一篇文章的配圖、或某個流程的流程圖（flowchart），就能拿到可手動微調、或直接貼進最終文件的向量圖。

SVG 圖表集
一篇文章所需的圖表都內嵌繪製，可以逐一微調、逐一複製出來。
10-svg-illustrations.html→
附註解的流程圖
把部署（deploy）管線畫成一張真正的流程圖，點任一步驟就能看到執行內容、耗時與失敗路徑。
13-flowchart-diagram.html→

簡報

1 個範例

幾個 <section> 標籤加上二十行 JS，就是一份簡報（slide deck）。把某個 Slack 訊息串或設計文件丟給代理，就能拿到在會議裡可用方向鍵翻頁的成品，不必開 Keynote、也不必匯出。

方向鍵簡報
一份簡短簡報打包成單一 HTML 檔。用左右鍵翻頁，不必經過 build。

07

研究與學習

2 個範例

一份有可摺疊區塊、分頁程式碼範本，以及邊欄術語表的說明頁（explainer），讀起來跟把同樣文字直直倒出來差很多。代理能搭出那層鷹架，讓一個新主題變得好上手。

一個功能是怎麼運作的
「說明這個 repo 的速率限制（rate limiting）」：附上一句話重點（TL;DR）框、可摺疊的請求路徑步驟、分頁的設定範本，以及常見問答。
14-research-feature-explainer.html→
概念說明頁
用一個可動態增刪節點的環來講一致性雜湊（consistent hashing），再配上對照表，以及滑鼠移過會連結的術語表。
15-research-concept-explainer.html→

報告

2 個範例

固定要出的文件，像狀態更新、事後檢討，最受益於一點結構與色彩。一張小圖和一條上色的時間軸，就能把大家只會掃過的東西，變成他們真的會讀的東西。

每週狀態
上了什麼、什麼延後了，配上一張小圖；排版設計成週一早上能快速掃過。
11-status-report.html→
事故時間軸
一份事後檢討：附上逐分鐘的時間軸、日誌節錄，以及後續追蹤清單。
12-incident-report.html→
09

自訂編輯介面

3 個範例

有時候很難在文字框裡描述你要什麼。那就跟代理要一個用完即丟的編輯器，專門處理你手上那件事；而且結尾永遠要有一個匯出按鈕，把你在介面裡做的所有操作轉回可貼回代理、或可 commit 的東西。你始終在迴圈裡，而且迴圈越轉越緊。

工單分流看板
把三十張工單在「現在 / 接下來 / 之後 / 砍掉」之間拖曳排列，再把最終順序複製成 markdown。
18-editor-triage-board.html→
功能旗標（feature flag）編輯器
切換開關依區域分組，前置條件關閉時會出現相依套件（dependency）警告，還有一個「複製差異」按鈕只複製改過的 key。
19-editor-feature-flags.html→
提示詞調校器（prompt tuner）
左邊是可編輯的範本，變數位置會標出來；右邊三組範例輸入會隨你打字即時重新渲染。
20-editor-prompt-tuner.html→
